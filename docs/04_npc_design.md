# 04 - NPC Design

## NPC Identity Rules

NPCs are Guild members, not static vending machines.

Each NPC requires:

- Three possible names
- Distinct personality
- Personal backstory
- Friendship progression
- Dialogue progression
- Optional personal questline

## Name Randomization

- At world creation, one name is selected from each NPC's three-name pool.
- Selected names persist for that world.
- This keeps replay variety while preserving consistency in one save.

## Trust and Friendship

- NPC Trust only increases.
- No trust loss mechanics.
- Trust tiers unlock dialogue depth, discounts, services, or personal quests.

## Core NPC Roster

- Guild Master
- Receptionist
- Weapon Shopkeeper
- Armor Shopkeeper
- Blacksmith
- Enchanter
- Magic Scholar
- Engineer
- Warehouse Keeper
- Tavern Host
- Cartographer
- Farmer
- Harbor Master
- Historian

## Dialogue Progression

- Stage 1: introduction and role explanation
- Stage 2: reacts to early Guild milestones
- Stage 3: reveals personal motives and concerns
- Stage 4: supports high-rank operations and legacy goals

## Personal Questline Guidelines

- Optional but meaningful
- 2 to 5 steps each
- Connect to world exploration and Guild growth
- Reward relationship progression, lore, or utility unlocks
