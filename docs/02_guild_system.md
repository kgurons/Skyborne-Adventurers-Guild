# 02 - Guild Core System

## Guild as a Shared Entity

The Guild is a persistent party-level progression object. All three players contribute to and benefit from the same:

- Guild Rank
- Guild Reputation
- Guild facilities
- Story progression milestones

## System Layers

- Social Layer: NPC relationships, tavern interactions, guild meetings.
- Operational Layer: Quest board, dispatch, emergency alerts.
- Engineering Layer: Airship research and component unlocks.
- Combat Layer: Dungeon and boss expedition planning.
- Legacy Layer: Hall of Heroes trophies, title records, historical milestones.

## Session Flow at HQ

1. Check guild notices and rank objectives.
2. Accept party quest set for the session.
3. Purchase and craft supplies.
4. Deploy by foot, portal, or airship.
5. Return, report outcomes, claim rewards, and unlock upgrades.

## Guild Facilities at Spawn

- Receptionist

- Armor Shop
- Blacksmith

- Magic Scholar
- Engineer
- Warehouse Keeper
- Tavern
- Cartographer
- Farmer
- Harbor Master
- Historian
- Hall of Heroes

## Design Rules

- Guild HQ must remain relevant at all game stages.
- Travel outwards should create motivation to return home.
- Rank-up moments should trigger visible world-state changes at HQ.
